# KAIKO　01. サービス概要

ver 1.1　|　2026年3月

---

## 1. サービスの目的

競馬の回顧情報を収集・管理し、ユーザーが馬単位・レース単位で情報を参照・入力できるWebサービスです。

YouTube字幕・競馬ブログ記事・オッズ情報を自動収集し、AIで構造化・整形して提供します。

「自分で回顧を掘るほどの時間はないが、本質的な予想がしたい」層に向けて、精査済みの分析情報を直感的なUIで届けることを目指します。

---

## 2. ターゲットユーザー

| 項目 | 内容 |
| --- | --- |
| 主要ターゲット | 競馬予想に本気だが、自分で回顧情報を収集・整理する時間・スキルがない中〜上級者 |
| ポジション | netkeiba（一般ファン向け）と JRA-VAN（超コア層向け）の中間層 |
| ペイン | 既存サービスは「使いやすいが浅い」か「深いが使いこなせない」の二択しかない |
| ゲイン | 精査済みの回顧情報を美しいUIで受け取り、自分で調べなくても本質的な予想ができる状態 |

## 3. サービスの強み

### 3-1. UI / UX 設計力

競合サービスはデータ量優先でUIが情報ダンプ型になっている。KAIKOはユーザーが情報を取得するまでのステップ数を最小化し、1画面で意思決定できる設計を徹底する。スマートフォン完全対応。

### 3-2. サービスとしての顔（馬比較・物差し馬）

- 馬ごとの本質的な能力を一目で把握できる馬ページ
- 物差し馬を軸にした馬と馬の相対比較機能
- 同一レースに出ていない馬同士でも、共通の基準馬を通じて能力を比較できる

この比較機能はどの競合も持っておらず、予想家が本当に知りたい「能力の相対感」を可視化できる唯一のポジション。

### 3-3. 精査済み回顧情報の価値

既存サービスには正しい回顧情報が存在しない。KAIKOは以下の構造で信頼性を担保する。

| レイヤー | 内容 |
| --- | --- |
| 収集 | YouTube字幕 ＋ 競馬ブログを同一レース・同一馬で自動収集・突合。複数ソースの一致が信頼性の根拠 |
| 構造化 | AIで回顧コメントを馬単位に整形。着順・状態・展開の軸で標準化 |
| 精査 | 開発者が確認した情報に「精査済み」マークを付与。信頼性を可視化 |
| 提供 | ユーザーには「KAIKOが整理・編集した分析情報」として提供 |

> ※ 将来構想：信頼できるユーザーへの限定UGC開放、ユーザーが自分の評価軸（展開・脚質・馬場など）でカスタム採点できる仕組みを検討。現フェーズには含まない。

### 3-4. ユーザー回顧の可視化・翻訳UI

ユーザーが自分で入力した回顧情報を、そのままメモとして溜めるのではなく、予想に使える形に整理・可視化することも強みの1つとして位置づける。

入力を効率化する仕組みはこちら側で提供しつつ、入力されたデータが馬単位・レース単位で自動的に構造化・集約されることで、蓄積するほど予想の精度が上がる体験を設計する。「自分の回顧が整理されているから使いやすい」という状態を作ることが価値の本質。

## 4. 主要機能

### 4-1. 自動収集機能

- YouTube字幕の自動取得（定期バッチ処理）
- 競馬ブログの回顧記事収集
- オッズ情報のリアルタイム収集

### 4-2. 参照・比較機能

- 馬名・レース名での検索
- 馬単位の回顧一覧表示
- 物差し馬を使った馬×馬の相対比較
- 精査済みマークによる情報信頼度の可視化
- スマートフォン対応（レスポンシブ）

### 4-3. ユーザー入力機能

- 馬単位の回顧情報入力フォーム
- レースID・馬IDによる情報管理
- ユーザー認証（ログイン・マイページ）

## 6. 将来構想 ― 予想の競技化

本サービスのターゲット層（競馬中〜上級者）にとって、競馬予想はギャンブルであると同時に「読みの精度を競う競技」でもある。現状の競馬情報サービスには、予想の実績を記録・検証・比較できる場が存在しない。有料予想家がSNSで実績を自己申告する構造が横行しているのも、事後検証の仕組みがないためである。

KAIKOはプロダクト内で予想を記録・スコアリングし、実績を客観的に可視化する場を提供することで、「予想の競技化」を実現したいと考えている。金銭的なリスクを一切伴わないポイント制により、純粋に予想精度を競える環境を目指す。

### 6-1. 競技化の設計方針（検討中）

何を「スコア」とするかは今後の検討事項だが、以下の軸が候補となる。

- 予想の的中率・回収率（結果ベースのスコア）
- 回顧の精度（事前の読みと事後の回顧がどれだけ一致したか）
- 上記の複合スコア（精度×再現性）

> ※ 「回顧の精度」軸はKAIKOのコア価値（回顧情報の蓄積）と最も親和性が高く、他サービスが提供できない独自スコアになりえる。

### 6-2. ランキングと特典設計

スコア上位ユーザーへの特典は、外部キャンペーン型とプロダクト内特権型の両立を想定する。

- **外部特典**：運営からのキャンペーン賞品・特別コンテンツ。マーケティング施策と連動し、ランキング発表をSNSでの拡散起点にする
- **プロダクト内特権**：回顧の精査権・優先表示・バッジなど。高スコアユーザーが自然と情報品質の担い手になる循環を作る

> ※ 本セクションはすべて将来構想。現フェーズ（Phase 1〜3）のスコープには含まない。具体的なスコア設計・特典内容・実施タイミングは別途検討する。
